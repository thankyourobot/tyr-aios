# Logic Model

How to map the theory of change for a mature sub-function.

## What It Is

A logic model traces how a sub-function produces its intended outcomes. It makes the theory of change explicit: given these resources, doing these activities, producing these outputs, we expect these outcomes.

The canonical structure:

**Resources → Activities → Outputs → Short-term Outcomes → Mid-term Outcomes → Long-term Outcomes**

The problem statement sits above this chain — the reason this sub-function exists at all.

## When to Use It

Logic models are for mature sub-functions, not new ones. The right question to ask: has this sub-function stabilized enough that "should it exist?" is settled, and the question is now "is it working as intended?"

If the sub-function is still being defined or regularly redesigned, a logic model adds overhead without insight. Apply it when the function is running and you want to evaluate and improve it systematically.

## How to Build One

Start by articulating the problem statement — what problem does this sub-function exist to solve? Then choose a direction:

**Forward (left to right):** Describe what the function does, then follow the chain to its outcomes. Useful when the function already exists and you're mapping what it produces.

**Backward (right to left):** Start with the desired long-term outcome, then work back to what resources and activities would produce it. Useful when designing a new sub-function or rethinking an existing one.

Either direction works. Treat it as iterative — the model will improve as you learn more.

## What Makes It Useful

A logic model earns its place when it changes what the agent does. If it produces measurable outcomes that feed back into the gap analysis (closing a gap, revealing a new one, or redefining what "excellent" looks like), it's working. If it's decorative, skip it.

The outputs of a logic model — specifically the outcomes — are natural candidates for KPIs that feed back into the meta-loop of the gap analysis.

## Relationship to Gap Analysis

The gap analysis identifies sub-functions and maps the territory at a high level. The logic model goes one layer deeper into a specific sub-function — mapping its internal theory of change. They work together: the gap analysis tells you *what* the domain needs to produce; the logic model tells you *how* a specific sub-function produces it.
